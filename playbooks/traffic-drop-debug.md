# Traffic Drop Debug Playbook

Use when the user asks why clicks, impressions or rankings fell.

## Required Spec

- affected date range;
- baseline date range;
- affected pages;
- affected queries;
- known deployments or content changes;
- Search Console data if available.

## Process

1. Separate clicks, impressions, CTR and average position.
2. Segment by page, query, country, device and Search appearance.
3. Check indexing and technical access.
4. Check SERP and demand changes.
5. Check content quality and competitive changes.
6. Compare timeline with site changes and public Google update windows.
7. Produce diagnosis with confidence levels.

## Output

- most likely causes;
- evidence;
- alternative causes;
- immediate checks;
- recovery plan;
- monitoring window.

## Caution

Do not blame an algorithm update without timeline and evidence.
